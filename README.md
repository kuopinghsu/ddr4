# DDR4 AXI4 Slave Simulation Model

A cycle-accurate, fully-parametrised **DDR4 AXI4 slave** simulation model written in SystemVerilog, designed to run under [Verilator](https://www.veripool.org/verilator/).

The primary goal is to give designers a realistic latency budget **before** connecting a real DDR4 controller — without requiring any proprietary DRAM IP or memory-controller hardware.

## Quick Start

```bash
# 1. Lint check
make bfm-lint

# 2. Run BFM testbench (DDR4-2400, AXI32, timing=1, N=50 random transactions)
make bfm-sim

# 3. Generate latency report across all speed grades, widths, and clock frequencies
make latency-report
```

## Make Targets

| Target | Description |
|---|---|
| `make bfm-lint` | Lint-only pass on BFM TB sources |
| `make bfm-sim` | Build & run BFM TB — 50 random txns per sequence (6 sequences) |
| `make bfm-cov` | Annotate functional coverage from last `bfm-sim` run |
| `make test-all` | Full BFM sweep: 7 speeds × 2 widths × 2 timing × 2 rand × 4 clocks = **224 combos** |
| `make latency-report` | Latency-focused sweep: 7 speeds × 2 widths × 4 clocks (timing=1, rand=0) = **56 combos**; writes `logs/latency_report.txt` and `logs/latency_report.csv` |
| `make clean` | Remove Verilator build artefacts (`obj_dir/`) |
| `make help` | List all targets |

### Overriding sweep axes (space-separated lists)

```bash
# Single speed, both widths, 1 GHz and 100 MHz only
make test-all SPEED_GRADES="2400" TIMING_MODELS="1" RANDOM_DELAYS="0" AXI_CLK_PERIODS="1 10"

# Custom latency report — 3 speeds, AXI32 only
make latency-report SPEED_GRADES="1600 2400 3200" AXI_WIDTHS="32"
```

## Key Parameters

### `ddr4_axi4_slave` module

| Parameter | Default | Description |
|---|---|---|
| `AXI_DATA_WIDTH` | 32 | AXI data bus width (32 or 64 bits) |
| `AXI_ADDR_WIDTH` | 32 | AXI address bus width |
| `DDR4_SPEED_GRADE` | 1600 | DDR4 data-rate in MT/s: 1600, 1866, 2133, 2400, 2666, 2933, 3200 |
| `DDR4_DENSITY_GB` | 1 | Memory density in GB (1–16) |
| `AXI_CLK_PERIOD_NS` | 10 | AXI clock (`aclk`) period in ns — e.g. 10 = 100 MHz |
| `ENABLE_TIMING_MODEL` | 1 | `1` = enforce real DDR4 tRCD/CL/CWL/tWR; `0` = bypass (1-cycle latency) |
| `RANDOM_DELAY_EN` | 0 | `1` = add random extra delays to model DDR4 page miss / refresh |
| `MAX_RANDOM_DELAY` | 10 | Maximum random extra delay in mclk cycles |
| `SIM_MEM_DEPTH` | 0 | Override memory depth (0 = use density-derived depth) |
| `BASE_ADDR` | `0x8000_0000` | AXI base address for this memory region |
| `VERBOSE_MODE` | 1 | `1` = print per-transaction messages |

### `ddr4_axi4_bfm_tb` top-level parameters (`-G` overrideable)

| Parameter | Default | Description |
|---|---|---|
| `DDR4_SPEED` | 2400 | Passed to `DDR4_SPEED_GRADE` |
| `AXI_DW` | 32 | AXI data width in bits |
| `ENABLE_TIMING` | 1 | Passed to `ENABLE_TIMING_MODEL` |
| `RANDOM_DELAY_EN` | 0 | Random delay enable |
| `CLK_PERIOD_NS` | 10 | AXI clock period in ns |
| `N_RAND` | 50 | Random transactions per sequence |
| `SIM_DEPTH` | 4096 | Simulation memory depth (words) |

## Architecture

```
                   ┌──────────────────────────────────────┐
                   │          ddr4_axi4_bfm_tb            │
                   │                                      │
                   │  AXI4 Master BFM ──► Scoreboard      │
                   │  (6 sequences × N_RAND txns)         │
                   │  Functional coverage (covergroup)    │
                   └──────────────┬───────────────────────┘
                                  │ AXI4
                   ┌──────────────▼───────────────────────┐
                   │         ddr4_axi4_slave              │
                   │                                      │
                   │  AXI clock domain (aclk)             │
                   │    Write FSM ──┐                     │
                   │    Read FSM  ──┤─► CDC handshake     │
                   │                │   (4-phase req/ack) │
                   │  DDR4 clock domain (mclk)            │
                   │    Timing countdown                  │
                   │    (tRCD + CL / CWL / tWR)           │
                   │                │                     │
                   │  Memory array ◄┘                     │
                   │  [0 : SIM_DEPTH-1]                   │
                   └──────────────────────────────────────┘
```

### Dual-clock CDC

The slave uses a **four-phase req/ack handshake** between the AXI clock domain (`aclk`) and the DDR4 memory clock domain (`mclk`). `mclk` runs at `DDR4_SPEED / 2` MHz (half-rate, matching the JEDEC double-data-rate convention). Latency timers fire in the `mclk` domain and are synchronised back to `aclk` — making the model accurate regardless of the ratio between `aclk` and `mclk`.

### Timing model

| Latency component | Modelled value |
|---|---|
| Row Activate → CAS (tRCD) | 14 ns (ceil to mclk cycles) |
| CAS Read Latency (CL) | 11–22 cycles, speed-grade dependent |
| CAS Write Latency (CWL) | 9–16 cycles, speed-grade dependent |
| Write Recovery (tWR) | 15 ns (ceil to mclk cycles) |
| **Total read latency** | tRCD + CL (mclk cycles) |
| **Total write latency** | tRCD + CWL + tWR (mclk cycles) |

## BFM Test Sequences

The BFM testbench exercises 6 randomised sequences, each running `N_RAND` transactions:

| Sequence | What it tests |
|---|---|
| `single_rw` | Single-beat INCR writes then reads back |
| `burst_incr` | INCR bursts, random length 1–15 |
| `burst_wrap` | WRAP bursts (power-of-2 lengths: 2, 4, 8, 16) |
| `burst_fixed` | FIXED burst — same address repeated |
| `strobe` | Partial-write byte strobes, verify only written bytes |
| `backpressure` | Read with random `rready` de-assertion |

All results are checked against a **shadow-memory scoreboard** with byte-level granularity. Mismatches are flagged with `[MISMATCH]` and the run exits with `$fatal` on failure.

## Latency Results Summary

See [`docs/latency_analysis.md`](docs/latency_analysis.md) for the full measured latency table and analysis.

**Quick reference** (ENABLE_TIMING=1, aclk=1 GHz, AXI32):

| DDR4 Speed | Avg Read | Avg Write |
|---|---|---|
| DDR4-1600 | 46.1 ns | 69.1 ns |
| DDR4-2400 | 44.0 ns | 63.5 ns |
| DDR4-3200 | 41.7 ns | 60.1 ns |

> AXI data width (32 vs 64-bit) does **not** affect latency — the bottleneck is the DDR4 timing domain, not bus throughput.
